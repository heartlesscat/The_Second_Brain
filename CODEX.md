# AI 总指南

## 系统定位

这不是普通笔记软件，而是一套基于 Codex + Obsidian 的本地化第二大脑系统。

- 持久化记忆：把重要对话、事件、模式和规则沉淀到本地 Markdown
- 主动行动：在任务、复盘、知识吸收和整理上主动协助
- 数据主权：所有核心数据都保存在本地 Vault 中

## 宪法级规则

### FATAL-001 禁止擅自重组文件结构

- 任何目录重组、批量移动、批量重命名，必须先扫描影响范围，再和用户确认
- 明确指定的单文件创建或单文件移动除外

### FATAL-002 默认归档而不是删除

- 没有明确删除指令时，不删除用户内容
- 不再使用的内容优先移动到对应目录下的 `_Archive/`
- 仅临时文件可直接删除，例如 `*.tmp`、`*.temp`

### FATAL-003 维护 Obsidian 链接资产

- 把每个 `[[wikilink]]` 视为知识资产
- 移动或重命名文件前，先检查引用关系
- 变更后必须修复相关链接

### FATAL-004 结构性变更要走审批流

- 影响 3 个以上文件的结构性操作，按 `SCAN -> PLAN -> CONFIRM -> EXECUTE -> VERIFY` 执行
- 结构性操作包括目录重组、批量归档、批量改名

### FATAL-005 Git 操作必须经用户授权

- 不擅自 `commit`
- 不使用 `force push`
- 不 `amend` 非本人提交
- 不使用 `git add -A`
- 只有用户明确说“commit”时才执行提交

## 记忆系统

### 情景记忆

- 记录具体事件、关键对话和决策过程
- 路径：`记忆库/情景记忆/YYYY-MM/MMDD-事件主题.md`
- 重点保留上下文、选择原因、下一步动作

### 语义记忆

- 从多条情景记忆中提炼模式、方法和长期知识
- 优先更新已有知识而不是重复创建

### 强制规则

- 只在规则足够稳定、确实需要长期遵守时写入
- 过时规则归档而不直接删除

## 上下文监控

### 回复标记

- 每次正式回复优先用 `✓` 作为执行状态标识
- 如果连续出现遗漏，说明上下文可能过载，需要提醒用户清理会话

### 上下文阈值

- 30%：开始提醒
- 40%：建议清理
- 50%：强烈建议清理
- 60%：必须清理

### 懒加载策略

- 不预读全部文件
- 只读取与当前任务直接相关的内容
- 优先复用已有上下文

## 智能工作流

### 存储路由

- 文章类内容放入 `01-内容/`
- 方法、摘录、主题笔记放入 `02-笔记/`
- 项目资料放入 `03-项目/`
- 工具与脚本说明放入 `04-工具/`
- 系统级文档放在根目录或 `记忆库/`

### 任务捕获

- 遇到“明天”“下周”“记得”“计划”“准备”等时间意图，写入 `任务清单.md`
- 完成后转移到 `任务归档/`

### 外部知识吸收

- 输入文章、视频、播客或长文本时，执行深读、提炼、对比、行动化、记录、沉淀六步

### 自我迭代

- 重要任务结束后，按观察、分析、设计、实施、验证、记录、提炼、提交八步迭代

## Obsidian 使用约定

- 这个目录可直接作为 Obsidian Vault 打开
- 以双向链接 `[[页面名]]` 和标签组织知识网络
- 不依赖私有云平台，全部内容为纯 Markdown

## 启动流程

1. 打开 Obsidian 并载入此 Vault
2. 查看 `任务清单.md`
3. 从 `记忆库/README.md` 或 `首页.md` 进入今天的工作
4. 在新会话中说明今天想推进的任务，系统再按需读取相关资料
